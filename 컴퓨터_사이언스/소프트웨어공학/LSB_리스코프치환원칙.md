# LSP (Liskov Substitution Principal)

- [LSP(Liskov Substitution Principle) : 리스코프 치환 원칙](https://server-engineer.tistory.com/226)

정확히 어떤 개념인지 머리 안에 제대로 정리가 되지 않았었는데 위 블로그 글을 읽고 확실히 감을 잡았다. LSP의 기본 개념은 '상위 타입의 객체를 하위 타입의 객체로 치환해도 상위 타입을 사용하는 프로그램은 정상적으로 동작해야 한다.' 이것이다. 이렇게만 놓고 보면 와닿지가 않는다.

위 블로그에서 든 예시를 보면 확실히 이해할 수 있다.  

```swift
let 차범근:차범근 = 차범근()
let 차두리:차범근 = 차두리() // 위반!!
```

차두리는 차범근의 아들이다.  하지만 그렇다고 해서 아빠는 아니다. 아빠의 유전자를 상속받은 것은 맞지만 아빠의 모든 유전자를 물려받은 것은 아니기 때문이다. 

차두리는 차범근의 모든 속성을 물려받지 못했다. 아들 객체가 아빠 타입의 모든 속성을 사용할 수 없다. 이것은 리스코프 치환 원칙의 위반이다. 

```swift
protocol 축구선수로서의자질 {
	...
}
class 차범근: 축구선수로서의자질 {
	...
}
class 차두리: 축구선수로서의자질 {
	...
}

let 차범근:축구선수로서의자질 = 차범근()
let 차두리:축구선수로서의자질 = 차두리()
```

이렇게 하면 상위 타입인 축구선수로서의자질의 모든 속성과 기능들을 하위 클래스인 차범근, 차두리가 사용할 수 있다. 이제 리스코프 치환 법칙을 만족한다.
